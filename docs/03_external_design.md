# 外部設計書（MVP・確定版）：ハッピーニュース（Android）

## 1. 目的・対象範囲
- Androidアプリで「ハッピーなニュースのみ」を毎日20本配信する
- MVP対象：一覧/詳細/保存/共有/設定/通知、API配信、日次バッチ
- MVP対象外：全文保存/全文翻訳、多言語UI、ユーザー投稿、高度パーソナライズ、課金（※将来拡張）

## 2. 前提・方針（外部仕様として明文化）
- 掲載：**20本/日**（JST基準）
- 要約：**日本語3行要約**（原文が日本語以外でもLLMで日本語要約を直接生成）
- 本文：本文保存は行わず **要約＋出典リンク**中心
- UX：B案（軽量サムネ）を採用し **1MB/日/ユーザー** を目標
- サムネ：**MVPは外部直リンク**（失敗時プレースホルダー、遅延ロード＋キャッシュ）
- 通知：**1日1回**、通知時刻は **時単位（HH:00）** を選択
- 著作権配慮：出典明記、リンク誘導、本文転載しない
- フォールバック：当日0件時は直近日を表示（最大7日）

## 5. 画面仕様

### SC-01 記事一覧（Today）
**目的**：当日の掲載記事を「軽く・気持ちよく」閲覧できること

- 表示：当日掲載 **20本**
- 並び順：**ハッピースコア降順**（同スコアは公開日降順）
- リスト項目：
  - サムネ（小・遅延ロード、外部直リンク）
  - タイトル
  - 3行要約（固定）
  - 出典名
  - 公開日
  - カテゴリ/タグ（1〜2個表示）
  - 保存アイコン（1タップ）
- 操作：
  - Pull to refresh（当日分を再取得）
  - ページング：MVPは不要（当日20本固定）

**フォールバック（重要）**
- 当日分が取得できない／0件の場合：
  - 直近日（latest day_key）の記事を表示
  - それも不可なら「取得できませんでした」＋再試行導線
- 遡り範囲：**最大7日**

**サムネ表示ルール（MVP確定）**
- 画像取得失敗時：プレースホルダー表示
- 画像は遅延ロード＋キャッシュ（B案の体感維持）

**将来（課金導入時）メモ**
- 無料：10本/日まで（例）／有料：20本/日フル＋広告OFF  
  ※課金導入時にSC-01に「続きを見る（プレミアム）」導線を追加する

---

### SC-02 記事詳細
- 表示：
  - タイトル
  - 日本語3行要約（または3段落）
  - 出典名・公開日
  - タグ（複数表示）
  - ボタン：保存/共有/出典を開く
- 操作：
  - 保存トグル
  - 共有（リンク＋定型文）
  - 出典リンクは外部ブラウザへ遷移

---

### SC-03 保存（ブックマーク）一覧
- 表示：保存記事（新しい順）
- 操作：
  - タップで詳細へ
  - 解除：スワイプ or ボタン（MVPはどちらでも可）
- エンプティ：保存がない場合のガイダンス表示

---

### SC-04 設定
- 設定項目（MVP）
  - 通知：ON/OFF、通知時刻（SC-05へ）
  - ミュート：**ミュートワード**（追加/削除）
  - 法務/情報（SC-06へ）

- ミュート仕様（明文化）
  - MVP：ミュートワードのみ
  - 将来：ミュートカテゴリ（実装難度とUXを見て追加）

---

### SC-05 通知設定（MVP確定）
- 通知ON/OFF
- 通知時刻：**時単位（HH:00）選択**（例：06:00〜10:00から選択、または0〜23時）
- OS権限未許可の場合：
  - 状態表示＋OS設定画面への誘導（「通知設定を開く」）

---

### SC-06 法務/情報
- 出典方針（要約＋リンク中心、本文保存しない）
- 利用規約/プライバシーポリシーへのリンク
- お問い合わせ（メール or フォーム）
- クレジット（必要に応じて）

---

## 6. 通知仕様
- 種類：デイリーダイジェスト（1日1回）
- 送信時刻：ユーザー設定（HH:00）
- 文面例：
  - タイトル：「今日のハッピーニュース」
  - 本文：「世界の良い出来事を20本まとめました」
- タップ時：SC-01（当日一覧）を開く
- フェールセーフ：当日0件の場合は直近日を表示

---

## 7. API仕様（外部IF）

### 7.1 day_key 仕様
- 形式：**YYYY-MM-DD（JST基準）**
- 例：`2026-03-01`

### 7.2 認証
- MVP：匿名（端末識別 or Firebase匿名認証）
- 目的：ブックマーク・通知設定の同期

### 7.3 エンドポイント一覧
- GET `/v1/days/latest`
  - 最新の掲載日（day_key）を返す

- GET `/v1/days/{day_key}/articles`
  - 指定日の掲載記事一覧（最大20本）
  - 並び順：happy_score desc → published_at desc
  - 当日0件時：latestへフォールバック

- GET `/v1/articles/{id}`
  - 記事詳細（要約・タグ・出典URL）

- GET `/v1/users/me/bookmarks`
  - 保存一覧

- POST `/v1/users/me/bookmarks/{articleId}`
  - 保存追加

- DELETE `/v1/users/me/bookmarks/{articleId}`
  - 保存解除

- PUT `/v1/users/me/settings`
  - 通知ON/OFF、通知時刻（HH:00）、ミュートワード等

### 7.4 レスポンス（例：記事一覧）
- ArticleSummary
  - id: string
  - title: string
  - summary_3lines: string
  - source_name: string
  - published_at: string (ISO8601)
  - thumbnail_url: string
  - tags: string[]
  - category: string
  - is_bookmarked: boolean
  - day_key: string (YYYY-MM-DD)

---

## 8. エラー設計
- ネットワーク不可：端末キャッシュ表示＋再試行
- 5xx：直近日フォールバック＋軽い案内
- 記事0件：直近日へフォールバック（最大7日）
- 権限（通知）：OS設定誘導
- レート制限：指数バックオフ＋簡易表示

---

## 9. データ保持とキャッシュ

### 9.1 端末側キャッシュ
- 当日一覧をローカルキャッシュ（起動体験優先）
- サムネ：HTTPキャッシュ＋ローカルキャッシュ（期限管理）
- 保存記事：ローカル永続化（オフラインで確認可能）

### 9.2 サーバ側保持（MVP確定）
- 掲載記事（20本/日）を確定保存
- 候補記事：**7日保持（全文なし、メタ＋抜粋＋判定結果）**

---

## 10. 監視・ログ（外部仕様としての運用要件）
- 監視対象：
  - バッチ成功/失敗
  - 掲載確定件数（20本/日が出たか）
  - NG混入検知（NGワード/カテゴリ）
  - 転送量の逸脱（B案の1MB/日/人からの乖離）
  - APIエラー率・レイテンシ
- アラート：
  - バッチ失敗、掲載0件、NG混入、コスト急増は通知対象

---

## 11. 非機能要件（外部仕様）
- 通信量：**1MB/日/ユーザー目標（B案）**
- パフォーマンス：一覧初回表示が軽いこと（画像遅延ロード）
- 可用性：バッチ失敗時も直近日を表示
- セキュリティ：認証情報の秘匿、レート制御
- 著作権：本文保存しない、要約＋リンク、出典明記
